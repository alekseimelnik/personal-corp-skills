# Фейбл руки-агенты

Ручной режим оркестрации для Claude Fable 5 в условиях урезанных лимитов: Fable сам не пишет и не читает код. Он пишет полные самодостаточные спеки в тела GitHub issues и раздаёт готовые задачи дешёвым субагентам — Sonnet на кодинг и анализ, Haiku на чтение и сверки.

## Ключевые правила

- **Issue-first, без исключений.** Любая задача — из бэклога или из чата — сначала становится issue, тело которого — полная спека (цель, контекст, решённые развилки, шаги, границы, DoD). Только потом диспатч.
- **Диспатч указателем.** Исполнитель читает тело issue через `gh issue view N`; промпт — только операционный конверт.
- **Отдельный верификатор на каждую задачу.** Субагент с чистым контекстом прогоняет проверку из DoD; принимает не тот, кто делал.
- **Лестница при провале:** две доработки тем же исполнителем → свежий исполнитель с диагнозом верификатора → лейбл `blocked` и короткий доклад пользователю.
- **Спекование вперёд.** Пока исполнитель работает, Fable пишет спеки следующих задач, а не ждёт.
- Параллельность определяется пересечением файлов, а не количеством агентов: задачи по одному файлу — последовательно с прямыми коммитами в main; файлово-независимые группы — параллельно в ворктри.

## Зачем

Токены Fable — самый дорогой ресурс сессии. Каждый tool call, который могла сделать модель дешевле, — сожжённый лимит. Скилл превращает Fable в чистого спекописателя и диспетчера.

Скилл вызывается вручную (`disable-model-invocation: true`), моделью автоматически не подтягивается.
