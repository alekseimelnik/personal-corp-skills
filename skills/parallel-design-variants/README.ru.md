# Parallel Design Variants Skill

Сгенерировать N по-настоящему разных дизайн-направлений сразу — каждое делает отдельный субагент, привязанный к своему визуальному референсу — собрать их в одну галерею, выбрать победителей и провести второй раунд, который смешивает выигравшие стили.

## Что это

Когда иначе ты сделал бы один дизайн и точил бы его, но направление ещё не выбрано — этот скилл превращает выбор в параллельный bake-off. Главная идея: **дивергенция рождается из референс-якорей, а не из просьбы «сделай покреативнее».** Десять субагентов с заданием «сделай красивый hero» дадут десять похожих hero; десять субагентов, привязанных к terminal / газете / брутализму / швейцарской сетке / зину, дадут десять разных миров с одним и тем же контентом.

## Когда применять

- Редизайн, лендинг, hero, раскладка обложки, письмо или система слайдов, где направление открыто
- Нужен реальный выбор между вариантами, а не доточка первой догадки
- Лайв/стрим или командный design bake-off, где за варианты голосуют

Не подходит:

- Когда направление одно и известно — просто сделай его
- Для пиксельной доводки одного выбранного дизайна — это обычная итерация

## Цикл

```text
спросить, сколько вариантов (N)
  → spec-документ (константный контент + N референс-якорей + брифы)
  → N субагентов параллельно (вслепую друг к другу)
  → галерея index.html (рядом, бок о бок)
  → выбор победителей (опционально голосованием)
  → второй раунд: микс выигравших стилей
```

## Контракт метода

- **Один spec-документ** держит всё: константный контент, раскладку результата, таблицу N референс-якорей, общий промпт-каркас и брифы по направлениям
- **Константный контент** — одинаковые реальные данные и обязательные секции во всех вариантах; различается только визуальная система
- **Спрашивает, сколько вариантов (N)** — предлагает 6–10; каждый привязан к отдельному узнаваемому референсу (смежные стили = потраченный вариант)
- **Самодостаточный HTML** — инлайн-CSS, ванильный JS, без билд-шага и фреймворков, чтобы галерея просто открывалась
- **Параллельный запуск** — субагенты вслепую друг к другу; именно эта независимость даёт дивергенцию
- **Конвергенция через микс** — второй раунд смешивает выигравшие стили, а не полирует один и не начинает заново

## Установка

```bash
cp -r skills/parallel-design-variants ~/.claude/skills/
```

После этого скилл доступен в Claude Code.

## Как вызвать

Скажи агенту:

> «Используй parallel-design-variants и дай варианты дизайна для этого hero».

Или:

> «Прогони design bake-off для лендинга — несколько разных направлений, потом смешай победителей».
